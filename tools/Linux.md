Linux常用命令

# 基础知识

Linux系统与Windows系统不同，没有盘符的概念。

例如Windows系统文件目录一般是：

- `D:\DevLog\user  、 C:\Java\development`

Linux系统的文件都是在根目录`/`下，例如`/user/local`

系统进入后默认在HOME（家）目录下，指的就是`/home/admin`这样的文件夹，一个Linux可以有多个用户登录，每个用户登录后都会又一个自己的家目录，通常是`/home/用户名`

命令结构`Command [-options] [parameter]`

- `command`：命令本身
- `-options`：[可选，非必填]命令的一些选项，可以通过选项控制命令的行为细节
- `-parameter`：[可选，非必填]命令的参数，多数用于命令的指向目标等
- `[]`语法中的，表示可选的意思

示例：

`ls -l/home/itheima`，`ls`是命令本身，`-l`是选项，`/home/itheima`是参数

- 意思是以列表的形式，显示/home/itheima目录内的内容

`cp -r test1 test2`，`cp`是命令本身，`-r`是选项，`test1`和`test2`是参数

意思是复制文件来`test1` 成为 `test2`

# 切换工作目录（cd）


Linux终端（命令行）打开的时候，会默认以用户的HOME目录作为当前的工作目录

可以通过`cd`命令，更改当前所在的工作目录。

`cd`命令来自英文：`Change Directory`

- 语法：`cd [Linux路径]`
- `cd`命令无需选项，只有参数，表示要切换到哪个目录下
- `cd`命令直接执行，**不写参数，表示回到用户的HOME目录**

# 查看文件`ls [-a -l -h] [查看文件夹地址]`命令

`ls`：列出当前目录下的文件信息（不包含隐藏文件），不加可选项一般平铺展开（例如，a文件 b文件 c文件）

`-a`：`all`的意思，展示所有文件信息包含隐藏文件（Linux中隐藏文件是以`.`开头的文件）

`-l`：`list`的意思，列表形式展示文件，展示包含文件详细信息，读写权限，用户，用户组，文件大小（不显示单位），创建日期

`-h`：展示文件详细信息时带有文件大小单位（kb，mb等），**必须搭配`-l`使用**

命令选项可以组合使用，例如`ls -l -a `、`ls -la `、` ls -al`

---

# 相对和绝对路径

例如目录`/home/admin`下有`Desktop`目录和`develop`目录，当前工作目录为`/home/admin`

- 绝对路径：以根目录`/`为起点，描述路径的一种写法，路径描述以`/`开头
- 相对路径：以当前目录为起点（目标目录需在当前目录下），描述路径的一种写法，路径描述无需以`/`开头

使用绝对路径切换：`cd /home/admin/develop`

使用相对路径切换：`cd develop`即可

---

# 特殊路径符

例如目录`/home/admin`下有`Desktop`目录和`develop`目录，当前工作目录为`/home/admin`

1. `.`：表示当前目录，比如`cd ./Desktop`表示切换到当前目录下的`Desktop`目录内，和`cd Desktop`效果一致
2. `..`：表示上一级目录，比如：`cd ..`即可切换到上一级目录，`cd ../..`切换到上二级的目录
3. `~`：表示HOME目录，比如：`cd ~`即可切换到`HOME`目录或`cd ~/Desktop`，切换到`HOME`内的`Desktop`目录

# 创建目录（mkdir）

**通过`mkdir`命令可以创建新的目录（文件夹）**

`mkdir`来自英文：Make Directory

语法：`mkdir [-p] Linux路径`

- 参数必填，表示Linux路径，即要创建的文件夹的路径，相对路径或绝对路径均可
- `-p`选项可选，表示自动创建不存在的父目录，适用于创建连续多层级的目录

**注意**：创建文件夹**需要修改权限**，请确保操作均在HOME目录内（**就是用户自己的家目录`/home/用户名`）**，不要**在HOME外操作涉及到权限问题**，HOME外无法成功**后续会补充权限管控的知识**

# 查看文件内容（cat / more）

**`cat`命令查看文件内容**

有了文件后，我们可以通过`cat`命令查看文件的内容

语法：`cat Linux文件路径`

- `cat`同样没有选项，只有必填参数，参数表示：被查看的文件路径，相对、绝对、特殊路径符都可以使用

---

**`more`命令同样可以查看文件内容**，同`cat`不同的是：

语法：`more Linux路径`

2. 通过`q`退出查看

# 复制文件或文件夹（cp）

**`cp`命令可以用于复制文件\文件夹，`cp`命令来自英文单词：`copy`**

语法：`cp [-r] 参数1 参数2`

1. `-r`选项，可选，用于复制文件夹使用，表示递归（就是文件夹下的内容一并递归复制）。
2. 参数1，`Linux`路径，表示被复制的文件或文件夹
3. 参数2，`Linux`路径，表示要复制去的地方

# 移动文件或文件夹（mv）

**`mv`命令可以用于移动文件\文件夹，`mv`命令来自英文单词：`move`**

语法：`mv 参数1 参数2`

1. 参数1，Linux路径，表示被移动的文件或文件夹
2. 参数2，Linux路径，表示要移动去的地方，如果目标不存在，则进行改名，确保目标存在

# 删除文件文件夹（rm）

`rm`命令用于删除文件或者文件夹，（remove）

语法： `rm [-r -f] 参数1 参数2.... 参数N`

- `-r`：与`cp`命令一样`-r`用于递归删除文件夹，若是删除空文件夹直接使用`rm`即可，若是文件夹里有内容就加`-r`递归删除
- `-f`：强制删除（force），不加会出现确认删除提示，加了之后不会弹出确认信息
- 参数1,2...N： 表示删除的文件或文件夹路径，可同时删除多个文件，逗号隔开

`rm`命令支持通配符`*`，用来做模糊匹配

符号`*`表示通配符，即匹配任意内容（包含空），示例：

`test*`：表示匹配任何以`test`开头的内容

`*test`：表示匹配任何以`test`结尾的内容

`*test*`：表示匹配任何包含`test`的内容

# 查找文件（find）

在Linux系统中，我们可以通过 find 命令去搜索指定的文件。

1. 按文件名查找：`find 起始路径 -name "被查找文件名"`
2. 按文件大小查找：`find 起始路径 -size +|- n[kMG]`
   - `+ -`表示大于和小于
   - `n`表示大小数字
   - `kMG`表示大小单位，k(小写字母)表示kb，M表示MB，G表示GB
     - 查找小于10KB的文件：`find / -size -10k`
     - 查找大于100MB的文件：`find /-size +100M`
     - 查找大于1GB的文件：`find /-size +1G`

**同样可以结合通配符`*`来进行模糊查找**

# 过滤命令（grep）

从文件内容中或者文件中进行过滤，得到我们需要查找的内容，（例如查看日志文件）

语法：`grep [-n] 关键字 文件路径`

- 选项`-n`：可选，表示在结果中显示匹配的行的行号。
- 参数，关键字：必填，表示过滤的关键字，带有空格或其它特殊符号，建议使用`""`将关键字包围起来
- 参数，文件路径：必填，表示要过滤内容的文件路径，可作为内容输入端口

---



# 数量统计命令（wc）

语法：`wc [-c -m -l -w] 文件路径`

1. 选项，-C：统计bytes数量
2. 选项，-m：统计字符数量
3. 选项，-L：统计行数
4. 选项，-W：统计单词数量
5. 参数，文件路径：被统计的文件，可作为**内容输入端口**（参考**管道符`|`**）

什么选项也不加直接使用 wc 默认统计行数，单词数，字节数

 # 管道符命令（|）

管道符的含义是：**将管道符左边命令的结果，作为右边命令的输入**

例如结合 grep命令使用：`cat test.txt | grep 要过滤的关键字`

作用是：把命令`cat test.txt`查看文件内容，并把内容作为输入参数传给命令 `grep 要过滤的关键字 cat test.txt`这样的形式一样，把查看到的文件内容传给 grep的文件路径位置，使用grep对查到的文件进行过滤 。

# 命令行输出指定内容（echo）

可以使用echo命令在命令行内输出指定内容，语法：`echo 输出的内容`

- 无需选项，只有一个参数，表示要输出的内容，复杂内容可以用`""`包围
- 带有空格或`\`等特殊符号，建议使用双引号包围,因为不包围的话，空格后很容易被识别为参数2，尽管echo不受影响，但会造成特殊符号不会被识别到，也就不会打印出来

## 反引号（`）

可以通过将命令用反引号将其包围被包围的内容，会被作为命令执行，而非普通字符

例如：

```
echo `pwd`  //会把pwd当命令执行而不是直接打印 pwd
```

同图形化界面中的文本编辑器一样，`vi`是命令行下对文本文件进行编辑的绝佳选择。

`vim`是`vi`的加强版本，兼容`vi`的所有指令，不仅能编辑文本，而且还具有`shell`程序编辑的功能，可以不同颜色的字体来辨别语法的正确性，极大方便了程序的设计和编辑性。

-  如果文件路径表示的文件不存在，那么此命令会用于编辑新文件

进入编辑功能后按 i 键进行编辑，编辑完成后按 ：冒号后输入 wq 保存并退出编辑模式

# 切换用户（su和 exit 命令）

`su`命令就是用于账户切换的系统命令，其来源英文单词：Switch User

语法：`su - [用户名]`

- `-`符号是可选的，表示是否在切换用户后加载环境变量（后续讲解），建议带上
- 参数:用户名，表示要切换的用户，用户名也可以省略，省略表示切换到 root
- 切换用户后，可以通过exit命令退回上一个用户，也可以使用快捷键：ctrl+d
- 使用普通用户，切换到其它用户需要输入密码，如切换到root用户使用root用户切换到其它用户，无需密码，可以直接切换

# 命令授权（sudo）

在我们得知 roo t密码的时候，可以通过 su 命令切换到root得到最大权限。但是我们不建议长期使用 root 用户，避免带来系统损坏。
我们可以使用 sudo 命令，为普通的命令授权，临时以 root 身份执行。

语法： `sudo 其它命令`

- 在其它命令之前，带上 sudo ，即可为这一条命令临时赋予 root 授权
- 但是并不是所有的用户，都有权利使用sudo，我们需要为普通用户配置sudo认证

## 为普通用户配置sudo认证

切换到 root 用户，执行`visudo`命令，会自动通过vi编辑器打开：`/etc/sudoers`（也可以使用`vim /etc/sudoers`）

在文件末尾添加：`用户名 ALL=(ALL)  NOPASSWORD: ALL`

其中最后的`NOPASSWD:ALL`表示使用 sudo 命令，无需输入密码。最后通过wq保存

# 用户用户组

Linux中关于权限的管控级别有2个级别，分别是：

- 针对用户的权限控制
- 针对用户组的权限控制

比如，针对某文件，可以控制用户的权限，也可以控制用户组的权限。用户组的权限是赋予该用户组所包含的用户

## 用户组管理

以下命令需root用户执行。

- 创建用户组`groupadd 用户组名`
- 删除用户组`groupdel 用户组名`

## 用户管理

以下命令需root用户执行。

- 创建用户

  `useradd [-g -d] 用户名`

  - 选项：-g指定用户的组，不指定-g，会创建同名组并自动加入，指定-g需要组已经存在，如已存在同名组，必须使用-g
  - 选项：-d指定用户HOME路径，不指定，HOME目录默认在：`/home/用户名`

- 删除用户

  `userdel [-r] 用户名`

  - 选项：-T，删除用户的HOME目录，不使用-r，删除用户时，HOME目录保留

- 查看用户所属组

  `id [用户名]`

  - 参数：用户名，被查看的用户，如果不提供则查看自身

- 修改用户所属组

  `usermod -aG 用户组 用户名`，将指定的用户加入到指定的用户组中

- 查看当前系统有哪些用户

  `gatent passwd`

# 认识文件或文件夹的权限信息

`rwx`：依次表示读，写，执行权限

`d`：表示是文件夹

`-`：表示是文件

例如`drwxr-xr-x`依次表示：

1. 对于第一位是`d`表示是文件夹，是`-`表示是文件
2. 第二到四位表示所属用户的权限
3. 第五到七位表示所属用户组的权限
4. 最后三位表示其他用户的权限
